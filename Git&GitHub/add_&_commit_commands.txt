Basic Commands:
~add: adds new or changed files in your working directory to the Git staging area.  (git add<-file name->)
~commit: it is the record of change  (git commit -m "some message")
~push: upload local repo content to remote repo   (git push origin main)

🔹 Basic Git Commands

add → Working directory ki new ya modified files ko staging area me le jaata hai

git add <file-name>

commit → Changes ka record/save point banata hai

git commit -m "message"

push → Local repository ke commits ko remote repository (GitHub) par upload karta hai

git push origin main
🔹 How to Finalize Changes?

👉 Jab hum project me:

New files create karte hain
Ya existing files me changes karte hain

To un changes ko final karne ke liye 2-step process hota hai:

Add → Commit
🔹 Step 1: Add (Staging Area)

👉 Chahe:

Tumne new file banayi ho
Ya purani file modify ki ho

Unhe working directory se staging area me le jaana hota hai.

📌 Flow:

Working Directory → Staging Area → Commit
✅ Example:
git add index.html

👉 Isse:

File staged ho jaati hai
git status me “Changes to be committed” me dikhai degi
🔹 Add All Files at Once

Agar sab files ek saath add karni ho:

git add .

👉 . ka matlab → current folder ki sab files

Check karne ke liye:

git status

👉 Output:

README.md
index.html
style.css
sab “Changes to be committed” me aa jaayengi
🔹 Real-Life Example (Letter)
Modified file = letter likh diya
git add = envelope me daal diya (ready hai)
git commit = stamp laga diya (final ho gaya)
🔹 Step 2: Commit

👉 Commit ka matlab hai:

Changes ka permanent record banana
✅ Syntax:
git commit -m "some message"

👉 Message me hamesha likho:

Tumne kya changes kiye
Example:
git commit -m "Added new files"
🔹 After Commit

Jab tum git status check karoge:

git status

👉 Output milega:

On branch main
Nothing to commit, working tree clean

✔️ Matlab:

Saare changes successfully save ho chuke hain
Files normal state me aa jaati hain (yellow → normal)
🔹 Important Points
Commit ek meaningful set of changes hona chahiye
Chhote-chhote changes ko add karke ek proper commit bana sakte ho
Git ab tumhare changes ko yaad rakhta hai (version control)
🔹 Next Step (Push)

👉 Ab local commits ko GitHub par bhejne ke liye:

git push origin main
