🔹 Push Command in Git

👉 Jo changes humne apne local system (computer) par kiye hain
(add + commit ke baad),
wo changes GitHub (remote repository) par bhi dikhne chahiye.

✔️ Iske liye hum push command ka use karte hain.

🔹 What is Push?

Push ka kaam:
Local repository ke saare commits ko remote repository (GitHub) par upload karna.

👉 Example:

System (local) → GitHub account (remote)
🔹 Syntax
git push origin main
🔹 Meaning of Each Term
git push → Local changes ko GitHub par bhejna (push karna)
origin → Remote repository ka naam (default name hota hai GitHub repo ka)
main → Branch ka naam jahan hum changes push kar rahe hain
🔹 First Time Push (Important)

👉 Jab tum first time push karte ho:

Ek popup / authorization window open hoti hai
Tumhe Allow / Authorize karna hota hai
VS Code ko GitHub access dena padta hai

✔️ Uske baad successfully push ho jaata hai

🔹 After Push

👉 GitHub par jaakar page refresh karo:

Jo files tumne VS Code me banayi thi:
index.html
style.css
README.md

✔️ Sab repository me dikhne lagenge (e.g., delta-demo folder ke andar)

🔹 Commit History on GitHub
GitHub par tumhe dikh jaayega:
Number of commits (e.g., 3 commits)
Last commit message (e.g., “Added new files”)
🔹 Language Detection

👉 GitHub automatically detect karta hai:

Tumne kaunsi language use ki hai

✔️ Example:

Agar tumne mostly HTML code likha hai
→ GitHub dikhayega 100% HTML
🔹 Simple Understanding

👉 git push =
“Apne local changes ko GitHub par bhejna (upload karna)”
