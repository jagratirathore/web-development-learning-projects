🔹 What is Forking?

👉 Forking ka matlab hota hai kisi aur ke GitHub repository ki copy apne GitHub account me banana

✔️ Ye ek new repository hoti hai
✔️ Ye original (upstream) repository se connected rehti hai

📌 Definition:

A fork is a new repository that is created from an existing repository and shares its code and visibility settings.

🔹 Simple Meaning

👉 Fork = Rough Copy / Duplicate (on GitHub account)

Tum kisi aur ka project directly edit nahi kar sakte
Isliye tum uska fork bana ke apne account me copy kar lete ho
Fir usme changes kar sakte ho
🔹 Example (Easy Understanding)

👉 Suppose kisi ne GitHub par Node.js project banaya hai

Tum us repo ko open karte ho
Tumhe Edit option nahi milega ❌
👉 Kyunki wo tumhara project nahi hai
🟢 Solution: Fork

👉 Tum Fork button par click karte ho

✔️ Result:

Same repository tumhare GitHub account me copy ho jaati hai
Same ya different name choose kar sakte ho

📌 Example:

Original Repo: nodejs-project
Forked Repo:   jagrati/nodejs-project
🔹 Fork ke baad kya kar sakte ho?

✔️ Code edit kar sakte ho
✔️ Files change kar sakte ho (e.g., README.md)
✔️ New features add kar sakte ho

📌 Example:

Edited by Jagrati Rathore
🔹 Pull Request (Fork ke baad)

👉 Agar tum apne changes original project me add karna chahte ho:

✔️ Tum Pull Request (PR) create karte ho

📌 Meaning:

Tum request bhejte ho original repo ke owner ko
Wo check karega (review)
Agar sahi laga → merge kar dega
🔹 Important Points
Fork → GitHub par hota hai (local nahi)
Fork ke baad hi tum dusron ke project me contribute kar sakte ho
Ye open-source contribution ka main process hai
🔹 Real-Life Analogy

👉 Fork = Rough Copy
👉 Original Repo = Fair Copy

✔️ Tum rough copy me changes karte ho
✔️ Fir final changes original me bhejte ho (PR ke through)

🔹 Final Summary
Fork → Copy repository to your GitHub account  
Edit → Make changes  
Pull Request → Send changes to original repo
